I studied the current Deriv OAuth flow and the key point is:

**Your callback URL must be registered exactly in your Deriv application settings.** If you want Deriv to return users to:

`https://digit-killer-001--ahmed-syn.replit.app/callback`

then that exact URL must be whitelisted in your Deriv OAuth app configuration. A mismatch (even a missing `/callback`) will fail. ([developers.deriv.com][1])

### Flow

1. User clicks **Login with Deriv**
2. Your app redirects to Deriv OAuth
3. User logs in on Deriv
4. Deriv redirects to:
   `https://digit-killer-001--ahmed-syn.replit.app/callback?code=XXXX&state=YYYY`
5. Your callback page extracts the `code`
6. Your backend exchanges the code for an access token
7. User is redirected to your dashboard/home page ([developers.deriv.com][1])

## Frontend Login Button (Replit)

```html
<button onclick="loginWithDeriv()">
    Login With Deriv
</button>

<script>
async function loginWithDeriv() {

    const clientId = "33rtqtfBfgRZqEpvayxel";
    const redirectUri = "https://digit-killer-001--ahmed-syn.replit.app/callback";

    const array = crypto.getRandomValues(new Uint8Array(64));

    const codeVerifier = Array.from(array)
      .map(v => 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789-._~'[v % 66])
      .join('');

    const hash = await crypto.subtle.digest(
      'SHA-256',
      new TextEncoder().encode(codeVerifier)
    );

    const codeChallenge = btoa(
      String.fromCharCode(...new Uint8Array(hash))
    )
    .replace(/\+/g, '-')
    .replace(/\//g, '_')
    .replace(/=+$/, '');

    const state = Math.random().toString(36).substring(2);

    sessionStorage.setItem("pkce_verifier", codeVerifier);
    sessionStorage.setItem("oauth_state", state);

    const authUrl =
      `https://auth.deriv.com/oauth2/auth` +
      `?response_type=code` +
      `&client_id=${clientId}` +
      `&redirect_uri=${encodeURIComponent(redirectUri)}` +
      `&scope=trade%20account_manage` +
      `&state=${state}` +
      `&code_challenge=${codeChallenge}` +
      `&code_challenge_method=S256`;

    window.location.href = authUrl;
}
</script>
```

Based on Deriv's OAuth documentation. ([developers.deriv.com][1])

## Callback Page (/callback)

```html
<!DOCTYPE html>
<html>
<head>
    <title>Deriv Callback</title>
</head>
<body>

<h2>Processing Login...</h2>

<script>
async function processCallback() {

    const params = new URLSearchParams(window.location.search);

    const code = params.get("code");
    const state = params.get("state");

    const savedState = sessionStorage.getItem("oauth_state");

    if (state !== savedState) {
        document.body.innerHTML = "State verification failed";
        return;
    }

    const verifier = sessionStorage.getItem("pkce_verifier");

    const response = await fetch("/exchange-token", {
        method: "POST",
        headers: {
            "Content-Type": "application/json"
        },
        body: JSON.stringify({
            code,
            verifier
        })
    });

    const data = await response.json();

    localStorage.setItem(
        "deriv_access_token",
        data.access_token
    );

    window.location.href = "/";
}

processCallback();
</script>

</body>
</html>
```

## Backend Endpoint (Node.js Express)

```javascript
app.post("/exchange-token", async (req, res) => {

    const { code, verifier } = req.body;

    const response = await fetch(
        "https://auth.deriv.com/oauth2/token",
        {
            method: "POST",
            headers: {
                "Content-Type":
                "application/x-www-form-urlencoded"
            },
            body: new URLSearchParams({
                grant_type: "authorization_code",
                client_id: "33rtqtfBfgRZqEpvayxel",
                code: code,
                code_verifier: verifier,
                redirect_uri:
                "https://digit-killer-001--ahmed-syn.replit.app/callback"
            })
        }
    );

    const tokenData = await response.json();

    res.json(tokenData);
});
```

This follows Deriv's authorization-code-with-PKCE flow. ([developers.deriv.com][1])

# 10-Line Prompt Explaining the Logic

```text
This application uses Deriv OAuth 2.0 with PKCE authentication.
When the user clicks Login, the app generates a code_verifier and code_challenge.
The user is redirected to Deriv's authorization server.
Deriv authenticates the user securely on its own website.
After successful login, Deriv redirects the user to the registered callback URL.
The callback URL receives an authorization code and state value.
The application verifies the returned state to prevent CSRF attacks.
The backend exchanges the authorization code for an access token.
The access token is stored securely and used for authenticated Deriv API requests.
Finally, the user is redirected back to the application's home page or dashboard.
```

### Important Check

Before using the code, verify in your Deriv Developer Dashboard that the registered redirect URI is exactly:

```text
https://digit-killer-001--ahmed-syn.replit.app/callback
```

If only the root URL is registered:

```text
https://digit-killer-001--ahmed-syn.replit.app
```

then Deriv will reject the OAuth request because the redirect URI must match exactly. ([developers.deriv.com][1])
